Cohort Retention Analysis (Online Retail)



Описание:

Цель проекта — проанализировать удержание клиентов с использованием когортного анализа и оценить, как пользователи возвращаются после первой покупки.



Инструменты:

SQL (PostgreSQL)
DBeaver
Excel (для визуализации)



Данные

Датасет содержит:

заказы (InvoiceNo)
даты покупок (InvoiceDate)
идентификаторы клиентов (CustomerID)
информацию о товарах и ценах



Этапы анализа:

1. Определение первой покупки клиента (когорта)
2. Присвоение каждому клиенту cohort\_month
3. Расчет месяца покупки (order\_month)
4. Вычисление “возраста” клиента в месяцах (month\_number)
5. Подсчет количества пользователей по когортам
6. Расчет retention rate (доли вернувшихся пользователей)



Основные выводы:

Retention по когортам находится примерно на одном уровне без существенного роста со временем
Наблюдается резкое снижение retention уже после первого месяца
Ранние когорты показывают более высокий уровень удержания по сравнению с последующими
Высокие значения retention на поздних этапах могут быть связаны с малым размером выборки



Интерпретация:

Основная потеря пользователей происходит после первой покупки
Продукт не формирует устойчивую привычку повторных покупок
Качество привлечения пользователей может снижаться со временем



Рекомендации:

Улучшить первый пользовательский опыт (onboarding)
Внедрить механики удержания после первой покупки (email, акции)
Анализировать каналы привлечения и их влияние на retention
Стимулировать повторные покупки через персонализированные предложения
